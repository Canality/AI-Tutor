## 🎯 功能逻辑变化概述

本次优化将 Instructor Agent 从**直接讲解式**教学转变为**苏格拉底式引导**教学，实现了教学理念的升级。

---

## 📊 详细对比

### 1. 教学理念变化

| 维度 | 优化前 | 优化后 |
|------|--------|--------|
| **核心方法** | 直接讲解答案 | 引导式提问 |
| **教学目标** | 让学生知道答案 | 让学生理解推导过程 |
| **交互模式** | 单向输出 | 双向对话 |
| **知识传递** | 一次性给全 | 分步骤引导 |

### 2. 输出结构变化

**优化前**：
```
自由文本格式
- 直接给出完整解答
- 学生被动接收
```

**优化后**：
```
💡 思路点拨（1-2句话说明核心逻辑）
📝 详细引导（具体推导，含LaTeX公式）
❓ 轮到你了（引导性问题，邀请用户思考）
✅ 最终解答（仅在最后一步出现）
```

### 3. 工作流程变化

**优化前**：
1. 理解问题
2. 检索知识（可选）
3. 直接给出完整解答

**优化后**（苏格拉底式5步法）：
1. **意图识别**：分析用户输入类型（新问题/回应/核对）
2. **知识检索**：按需调用工具，避免过度检索
3. **分步引导**：每次只聚焦一个关键步骤
4. **错误诊断**：指出逻辑漏洞，给出小提示（Hint）
5. **总结核对**：仅在最后输出完整解答

### 4. 错误处理变化

**优化前**：
- 学生回答错误 → 直接说"错了"
- 给出正确答案

**优化后**：
- 学生回答错误 → 指出具体逻辑漏洞
- 给出更小的提示（Hint）
- 引导学生自己发现错误

### 5. 工具使用变化

**优化前**：
- 工具调用规范较模糊
- 可能过度检索

**优化后**：
- 明确"按需调用，避免过度检索"
- 同一问题每个工具最多1-2次
- 已掌握知识直接引导，不调用工具

## 🎨 教学效果对比

### 场景1：学生询问通项公式

**优化前**：
```
学生：怎么求等差数列通项公式？
AI：等差数列通项公式是 an = a1 + (n-1)d。
推导过程：...
（一次性给出所有内容）
```

**优化后**：
```
学生：怎么求等差数列通项公式？
AI：
💡 思路点拨：等差数列的特点是相邻两项的差是常数。
📝 详细引导：我们已知首项 a₁，公差 d，要求第 n 项 aₙ。
   观察规律：
   - 第1项：a₁
   - 第2项：a₁ + d
   - 第3项：a₁ + 2d
   你发现规律了吗？
❓ 轮到你了：你能写出第 n 项的表达式吗？试着用 a₁、d、n 表示。
```

### 场景2：学生回答错误

**优化前**：
```
学生：q = 2
AI：错了，应该是 q = ±2。
```

**优化后**：
```
学生：q = 2
AI：
💡 思路点拨：再仔细检查一下计算过程。
📝 详细引导：根据 a₃ = a₁ × q²，我们有 4 = 1 × q²，所以 q² = 4。
   注意：一个正数有两个平方根。
❓ 轮到你了：4 的平方根有几个？分别是多少？
```

### 场景3：学生完成步骤

**优化前**：
```
学生：我算出来 q = 2 或 q = -2
AI：好的，下一步是...
（直接继续）
```

**优化后**：
```
学生：我算出来 q = 2 或 q = -2
AI：
💡 思路点拨：很好！你考虑到了正负两种情况。
📝 详细引导：q = 2 时，数列递增；q = -2 时，数列正负交替。
   两种情况下通项公式都成立。
❓ 轮到你了：现在你能写出通项公式了吗？分别写出 q=2 和 q=-2 的情况。
```

## 💻 技术实现

### 与项目架构的兼容性

| 方面 | 说明 |
|------|------|
| **代码修改** | 仅修改 `prompt.py`，不修改其他文件 |
| **工具调用** | 保持 `search_knowledge`, `search_examples` 不变 |
| **参数传递** | 保持 `input`, `chat_history`, `agent_scratchpad` 不变 |
| **Agent 类型** | 保持 `create_openai_tools_agent` + `AgentExecutor` |
| **流式输出** | 完全兼容 `solve_stream` 方法 |

### Prompt 结构

```python
INSTRUCTOR_PROMPT = ChatPromptTemplate.from_messages([
    ("system", "..."),  # ← 仅修改这里的内容
    MessagesPlaceholder(variable_name="chat_history"),
    ("human", "{input}"),
    MessagesPlaceholder(variable_name="agent_scratchpad"),
])
```
