# AI Tutor - 五人团队分工计划

## 文档信息
| 项目 | 内容 |
|------|------|
| 文档版本 | V1.0 |
| 团队规模 | 5人 |
| 文档目的 | 明确团队成员分工、职责与协作方式 |

---

## 一、团队成员角色分配

| 成员 | 角色 | 核心职责 | 主要负责模块 |
|------|------|----------|--------------|
| **成员A** | 前端开发工程师 | 前端页面开发、用户交互、API对接 | 前端模块 |
| **成员B** | 后端架构师 | 基础框架搭建、数据库设计、技术选型 | 基础框架模块 |
| **成员C** | 后端开发工程师 | 用户认证、API接口、数据模型 | 用户认证模块 |
| **成员D** | AI/后端开发工程师 | Instructor Agent、RAG、知识图谱 | Instructor Agent 模块 |
| **成员E** | AI/后端开发工程师 | Advisor Agent、推荐算法、学习画像 | Advisor Agent 模块 |

---

## 二、各成员详细职责

### 成员A - 前端开发工程师

**核心职责**：
- 负责所有前端页面和组件的开发
- 实现用户交互逻辑
- 对接后端 API 接口
- 前端样式与用户体验优化

**负责模块**：
- 前端模块（完整负责）

**阶段任务分解**：

| 阶段 | 主要任务 | 交付物 |
|------|----------|--------|
| **阶段 1**  | 1. 前端项目初始化<br>2. 登录/注册页面开发<br>3. 基础聊天页面开发<br>4. 对接认证和基础聊天 API | 可运行的前端原型 |
| **阶段 2-4**  | 1. 持续跟进后端 API 变化<br>2. 优化现有页面交互<br>3. 准备复杂页面的开发方案 | 前端阶段性优化版本 |
| **阶段 5**  | 1. 聊天/解题页面完善（支持流式输出）<br>2. 学习画像页面开发<br>3. 练习题推荐页面开发<br>4. 图片上传组件开发<br>5. 完整前后端联调<br>6. 前端测试与 Bug 修复 | 完整前端应用 |

**技术栈要求**：
- Vue3 (Composition API)
- Axios (HTTP 客户端)
- SSE (Server-Sent Events)
- CSS/SCSS (样式)

**协作对象**：
- 所有后端成员（API 对接）
- 成员B（API 规范约定）

---

### 成员B - 后端架构师

**核心职责**：
- 项目整体技术架构设计
- 后端基础框架搭建
- 数据库设计与配置
- 技术选型与规范制定
- 代码质量把控

**负责模块**：
- 基础框架模块（完整负责）
- 项目整体架构协调

**阶段任务分解**：

| 阶段 | 主要任务 | 交付物 |
|------|----------|--------|
| **阶段 1**  | 1. 项目目录结构搭建<br>2. FastAPI 应用初始化<br>3. MySQL 数据库设计与建表<br>4. Redis 配置<br>5. 配置管理系统（环境变量、config.py）<br>6. 日志系统搭建<br>7. CORS 等基础中间件配置<br>8. 制定 API 规范与开发规范<br>9. 编写 requirements.txt | 完整的基础框架、数据库初始化脚本、开发规范文档 |
| **阶段 2-5**  | 1. 持续优化基础框架<br>2. 解决架构层面问题<br>3. 代码评审<br>4. 协调各模块集成<br>5. 性能优化指导 | 架构优化记录、代码评审意见 |

**技术栈要求**：
- FastAPI
- SQLAlchemy (async)
- MySQL
- Redis
- Pydantic

**协作对象**：
- 所有后端成员（架构指导、规范执行）
- 成员A（API 规范约定）

---

### 成员C - 后端开发工程师

- 用户认证模块开发

- 数据库模型定义

- 用户认证模块（完整负责）

| 阶段 | 主要任务 | 交付物 |

## 三、关键依赖关系

```
成员B（基础框架）
    ↓
成员C（用户认证）←→ 成员A（前端）
    ↓
成员D（Instructor Agent）←→ 成员A（前端）
    ↓
成员E（Advisor Agent）←→ 成员A（前端）
```

**说明**：
- 成员B的工作是其他后端成员的基础
- 成员C完成认证模块后，成员A可开始对接
- 成员D和E可并行开发，但都依赖基础框架
- 成员A持续跟进后端API进展，及时对接

## 五、风险管理与应对

| 风险 | 涉及成员 | 应对措施 |
|------|----------|----------|
| 基础框架延期 | 成员B | 优先完成核心功能，非核心功能后续补充 |
| 前端进度滞后 | 成员A | 优先实现核心页面，简化非关键功能 |
| Agent 效果不理想 | 成员D/E | 快速迭代，先保证可用，再逐步优化 |
| 前后端联调问题多 | 全体 | 提前约定API规范，尽早开展联调 |
| 人员临时缺席 | 全体 | 代码文档化，关键模块双人备份 |
