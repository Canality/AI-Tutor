## 一、项目阶段规划

根据 README.md 中的阶段目标，将项目划分为 5 个主要开发阶段：

| 阶段 | 目标描述 | 预计工期 | 主要交付物 |
|------|----------|----------|------------|
| **阶段 1** | 基础框架搭建 + AI Tutor 问答系统初版 | 2周 | 可运行问答原型、静态题库测试示例 |
| **阶段 2** | 构建结构化题库 + RAG 模块 | 2周 | 小规模题库(100-200题)、向量化索引、RAG 增强的 Agent |
| **阶段 3** | 知识图谱构建 + 分步骤解题 | 2周 | KG 初版、Instructor Agent 可生成分步骤解题 |
| **阶段 4** | 个性化推送 + 学生画像 | 1.5周 | Advisor Agent 可推荐练习、学习画像生成 |
| **阶段 5** | 前端界面 + 前后端联调 | 1.5周 | 完整网页交互界面 |

**总工期**：约 9 周

---

## 二、详细开发计划

### 阶段 1：基础框架 + 问答系统初版（第 1-2 周）

**目标**：
- 完成项目基础框架搭建
- 实现无 RAG 和 KG 的初版问答原型

**主要任务**：
1. 项目初始化与环境配置
2. 数据库设计与建表
3. 基础 API 框架搭建
4. 用户认证模块（登录/注册）
5. Instructor Agent 初版（直接调用 LLM）
6. 前端基础页面（登录页、简单聊天页）

**里程碑**：
- Week 1 结束：基础框架完成，数据库可连接，用户可登录注册
- Week 2 结束：初版问答原型可用，能回答简单数列问题

---

### 阶段 2：结构化题库 + RAG 模块（第 3-4 周）

**目标**：
- 构建数学数列结构化题库
- 完成向量数据库搭建
- Agent 集成 RAG 模块

**主要任务**：
1. 整理/生成 100-200 道数列题目
2. 题目数据结构化（标注知识点、难度等）
3. 向量数据库（Chroma）搭建与配置
4. 题目向量化与索引构建
5. RAG 检索模块开发
6. Instructor Agent 集成 RAG

**里程碑**：
- Week 3 结束：题库整理完成，向量数据库可正常工作
- Week 4 结束：Agent 能通过 RAG 检索相关题目并生成答案

---

### 阶段 3：知识图谱 + 分步骤解题（第 5-6 周）

**目标**：
- 构建数列知识图谱
- 实现分步骤解题功能

**主要任务**：
1. 数列知识点体系梳理
2. 知识图谱（NetworkX）构建
3. 题目-知识点关联
4. 分步骤解题 Prompt 设计
5. Instructor Agent 生成分步骤解答
6. 知识图谱查询接口

**里程碑**：
- Week 5 结束：知识图谱初版完成，知识点关联建立
- Week 6 结束：Instructor Agent 可提供清晰的分步骤解题讲解

---

### 阶段 4：个性化推送 + 学生画像（第 7-8 周上半周）

**目标**：
- 实现学生学习画像
- 个性化练习题推送
- 短期学习计划生成

**主要任务**：
1. 学习记录数据模型
2. 学生画像计算逻辑
3. 推荐算法实现（基于错题/薄弱点）
4. 学习计划生成逻辑
5. Advisor Agent 开发
6. 相关 API 接口

**里程碑**：
- Week 7 结束：学生画像可生成，推荐逻辑初步可用
- Week 8 中期：Advisor Agent 可推荐练习和生成学习计划

---

### 阶段 5：前端界面 + 前后端联调（第 8 周下半周 - 第 9 周）

**目标**：
- 完成所有前端页面
- 前后端接口联调
- 系统整体测试

**主要任务**：
1. 聊天/解题页面完善
2. 学习画像页面
3. 练习题推荐页面
4. 多模态输入（图片上传）
5. SSE 流式输出集成
6. 前后端完整联调
7. 系统测试与 Bug 修复

**里程碑**：
- Week 8 结束：主要前端页面完成
- Week 9 结束：系统完整可用，可交付演示

## 三、风险管理

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| LLM API 调用成本过高 | 高 | 中 | 优化 Prompt，设置调用上限，考虑本地模型 |
| 向量检索效果不理想 | 中 | 中 | 调优 Embedding 模型，优化检索策略 |
| 知识图谱构建工作量大 | 中 | 高 | 先做最小可用版，后续迭代完善 |
| 前端开发进度滞后 | 中 | 中 | 优先实现核心功能，简化非关键页面 |
| 前后端联调问题多 | 中 | 中 | 提前约定 API 规范，尽早联调 |

## 四、验收标准

### 阶段 1 验收
- [ ] 项目可正常启动
- [ ] 用户可注册/登录
- [ ] 可以输入数学问题并获得解答
- [ ] 解答内容基本正确

### 阶段 2 验收
- [ ] 题库中至少有 100 道数列题
- [ ] 题目包含知识点、难度等标注
- [ ] RAG 能检索到相关题目
- [ ] 解答质量相比阶段 1 有提升

### 阶段 3 验收
- [ ] 知识图谱包含主要数列知识点
- [ ] 题目与知识点正确关联
- [ ] 解答包含清晰的分步骤
- [ ] 步骤中提及相关知识点

### 阶段 4 验收
- [ ] 学生画像能展示知识点掌握度
- [ ] 能识别薄弱知识点
- [ ] 推荐的练习题与薄弱点相关
- [ ] 学习计划合理可行

### 阶段 5 验收
- [ ] 所有页面功能正常
- [ ] 支持图片题目输入
- [ ] 解题步骤流式展示
- [ ] 整体流程顺畅无明显 Bug

## 五、后续优化方向

1. **功能增强**：
   - 错题本功能
   - 更多题型支持
   - 学习社区/讨论区

2. **性能优化**：
   - 响应速度优化
   - 缓存策略优化
   - 并发处理优化

3. **技术升级**：
   - 接入更强大的 LLM
   - 优化 RAG 检索策略
   - 知识图谱自动扩展
